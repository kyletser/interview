## （算法）笔试前看：

1. **看数据范围**

   - n ≤ 20：直接 **回溯**
   - n ≤ 1000：**O(n²)** 随便写（DP、暴力）
   - n ≤ 1e5：必须 **O(n) / O(n log n)**（双指针、滑窗、单调栈、贪心）

2. **看题目关键词**

   - 子数组、连续：滑窗 / 前缀和
   - 子序列：DP
   - 组合、选择、不能重复：回溯 / 01 背包
   - 最大最小、最近更大：单调栈
   - 树：递归（前中后层序）
   - 图：BFS/DFS

3. **先想暴力解法**

   哪怕 O (n²) 也要先写出来，**有代码就有分**。

4. **再想怎么优化**

   暴力能过就过，不能过再说优化。

## 1）先问自己：能不能暴力枚举？

- 数组：两层 for 枚举所有区间
- 字符串：枚举起点终点
- 树：递归遍历

## 2）再问：能不能用哈希表存一下？

凡是要**找值、查存在、快速定位**，无脑想 `unordered_map` / `unordered_set`

例：两数之和、长度最小子数组、重复元素等。

## 3）再问：能不能排序？

排序后很多题直接变简单：

- 双指针
- 二分
- 去重
- 贪心

## 4）再问：能不能双指针？

满足下面任意一条就试：

- 数组有序
- 要找两个数
- 要 “夹逼”
- 要滑动窗口

## 5）再问：是不是 DP？

满足下面任意一条就试：

- 求最大 / 最小
- 求方案数
- 不能贪心（局部最优≠全局最优）
- 有选择：选 / 不选

## 6）再问：是不是回溯？

- 组合、排列、子集
- n 很小

## 7）实在不会：

**写 BFS / DFS 暴力搜，总能骗到步骤分。**



你可以直接说：

> 这道题我先想一个暴力思路，时间复杂度 O (n²)，虽然不是最优，但可以先实现出来，再慢慢优化。

面试官 99% 会点头，**比你僵住强一万倍**。
